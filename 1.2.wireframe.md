# [Wireframe] Oh My Cap (omc) 플러그인 UI/UX 설계도

본 문서는 PRD v7.0을 바탕으로 IntelliJ IDEA 내부에 통합될 `Oh My Cap` 플러그인의 시각적 구조와 레이아웃(Wireframe)을 정의합니다. 

## 1. 메인 채팅 화면 (Tool Window)
IntelliJ 우측(또는 하단) 패널에 도킹되어 상시 노출되는 핵심 에이전틱 컨트롤 타워입니다.

### 1-1. 기본 대화 및 액션 상태
```text
+-------------------------------------------------------+
| [🧢 Oh My Cap]  [👤 Backend Persona]         [⚙️ 설정] | <- 헤더 영역
+-------------------------------------------------------+
| 🟢 [Token Usage: 14K / 32K] ━━━━━━━━━━━━━━━━╸         | <- 토큰 인디케이터
+-------------------------------------------------------+
| 📁 History & Task Management                          |
|  [+ 새 대화 (New Task 시작)]                          |
|  └─ 🕒 [OrderController.java 리팩토링...]             | <- 히스토리 서랍 (복원)
|  └─ 🕒 [JIRA-4812 결제 오류 수정 건...]               |
+-------------------------------------------------------+
| [AI] JIRA-4812 티켓 분석을 완료했습니다.              |
|      수정 제안: OrderMapper.xml, OrderService.java    |
|                                                       | <- 동적 대화방
| ╭───────────────────────────────────────────────────╮ |
| │ 📝 OrderService.java (Diff View)                  │ |
| │ - int currentStatus = 0;                          │ |
| │ + int currentStatus = OrderStatus.PENDING;        │ |
| │ [ ✅ Apply (승인) ]  [ ❌ Ignore (거절) ]         │ |
| ╰───────────────────────────────────────────────────╯ |
|                                                       |
| [User] 단위 테스트 코드 구동 후 문제 없으면 캡쳐해 줘.|

+-------------------------------------------------------+
| ╭───────────────────────────────────────────────────╮ | <- 입력 프롬프트 영역
| │ [ Plan 모드(기획/검토) ] ↕️ [ Act 모드(실행) ]    │ |
| ├───────────────────────────────────────────────────┤ |
| │ @UserEntity.java (컨텍스트 마운팅)                │ |
```

### 1-2. 컨텍스트 메뉴 액션 연동 상태 (예: '코드 문맥 설명' 실행 시)
에디터에서 소스코드를 드래그 한 뒤 우클릭하여 `Explain Code Semantics` 액션을 눌렀을 때, 채팅창이 블록을 넘겨받아 즉시 반응하는 연계 예시입니다.
```text
+-------------------------------------------------------+
| [🧢 Oh My Cap]  [👤 Requirements Analyst]    [⚙️ 설정] |
+-------------------------------------------------------+
| 🟢 [Token Usage: 16K / 32K] ━━━━━━━━━━━━━━━━━━╸       |
+-------------------------------------------------------+
| [User] (Action Triggered: 🔍 Explain Code Semantics)  |
| ╭───────────────────────────────────────────────────╮ |
| │ @selected_code (OrderService.java: 145-160)       │ |
| │ // calculateTotalDiscount() 메서드 구현부...      │ |
| ╰───────────────────────────────────────────────────╯ |
|                                                       |
| [AI] 이 코드 블록의 동작 문맥(Context) 보고서입니다:  |
|                                                       |
| 1️⃣ **기능 요약:** 주문 객체의 총 할인액을 계산하는    |
|    VVIP 전용 계산기 로직 파트입니다.                  |
| 2️⃣ **의존성 파악:** `CouponValidator`, `GradePolicy`  |
|    객체와 강결합되어 유지보수 난이도가 높습니다.      |
| 3️⃣ **주의 사항:** NullPointer 발생 가능성이 있는       |
|    `coupon.getRate()` 호출부가 존재합니다.            |
|                                                       |
| 💡 **[후속 액션 추천] (Quick Actions)**                 |
|   [ 🧪 단위 테스트 생성 ]  [ 🛠️ Null-safe 리팩토링 ]  |
+-------------------------------------------------------+
| ╭───────────────────────────────────────────────────╮ |
| │ [ Plan 모드 ] ↕️ [ Act 모드 ]                      │ |
| ├───────────────────────────────────────────────────┤ |
| │ 저기서 Null 처리는 공통 Utils를 사용해서 다시 짜줘│ |
| ├───────────────────────────────────────────────────┤ |
| │ [↩️ Undo] [🛑 Stop]                 [🚀 전송(Send)]│ |
| ╰───────────────────────────────────────────────────╯ |
+-------------------------------------------------------+
```

## 2. 컨텍스트 메뉴 (Context Menu - 우클릭 액션)
에디터 화면 내부에서 코드를 드래그(`Block`)한 뒤, 마우스 오른쪽 버튼을 클릭했을 때 기본 IDE 메뉴 사이에 노출되는 플러그인 전용 확장 메뉴입니다.

```text
+-------------------------------------------------------+
| Cut                                            Cmd+X  |
| Copy                                           Cmd+C  |
| Paste                                          Cmd+V  |
|-------------------------------------------------------|
| 🧢 Oh My Cap (omc) Actions ───────────▶               |
|    ├─ 💬 Add Snippet to Chat (채팅으로 보내기)        |
|    ├─ 🔍 Explain Code Semantics (코드 문맥 설명)      |
|    ├─ ⚖️ Review Code Quality (코드 품질 평가)        |
|    ├─ 🛠️ Refactor Code Snippet (리팩토링 제안)       |
|    ├─ 🛡️ Inspect Static Analysis [PMD] (정적 검사)   |
|    ├─ 🌐 Analyze Impact Scope (영향 범위 분석)        |
|    ├─ 🧪 Generate Unit Tests (단위 테스트 생성)       |
|    ├─ 💾 Validate SQL Query (SQL 정합성 검증)         |
|    └─ 🔐 Scan Security Vulnerability (보안 체크)      |
|-------------------------------------------------------|
| Refactor                                              |
| Find Usages                                           |
+-------------------------------------------------------+
```

## 3. 환경 설정 패널 (Preferences / Settings)
IntelliJ의 기본 환경설정 단축키(`Cmd + ,` 또는 `Ctrl + Alt + S`)를 눌렀을 때 좌측 트리에 나타나는 전용 설정창입니다. 상단에 5개의 탭 버튼으로 구성됩니다.

```text
| [LLM Server]  [MCP]  [Persona]  [Prompts]  [General]                            |
+---------------------------------------------------------------------------------+
| 🎫 사내 이슈 트래커 연동 (Model Context Protocol)                               |
|                                                                                 |
|  * 연동 플랫폼:  [ (O) JIRA   ( ) Redmine   ( ) GitLab Issues ]                 |
|  * Base URL:     [ http://jira.company.internal/  ]                             |
|  * Project Key:  [ OMC_DEV_TEAM                   ]                             |
|  * Auth Token:   [ ****************************** ]                             |
|                                                                                 |
|  [ 🔗 Connection Test (인증 및 연결 테스트) ] -> ✅ 연결 성공! (Ping: 12ms)     |
|                                                                                 |
|  📋 검색 옵션 (Fetching Options)                                                |
|  * [☑️ 활성] 티켓 번호 호출 시 'Description' 본문 자동 Fetch                    |
|  * [☑️ 활성] 티켓에 등록된 최신 첨부파일(PDF/Excel) 자동 RAG 인덱싱             |
|                                                                                 |
|                                                     [ Apply ] [ OK ] [ Cancel ] |
+---------------------------------------------------------------------------------+
```

### 3-4. [Prompts] 탭 (매크로 시스템)
에디터에서 '우클릭 메뉴 단축 액션'을 실행할 때, AI 백엔드로 은밀하게 발송되는 프롬프트 문자열을 커스텀.
```text
+---------------------------------------------------------------------------------+
| [LLM Server]  [MCP]  [Persona]  [Prompts]  [General]                            |
+---------------------------------------------------------------------------------+
| 🖱️ 단축 액션용 프롬프트 매크로 (Context Menu Action Prompts)                    |
|                                                                                 |
| [ 우클릭 카테고리 (Actions) ]  [ 📝 해당 액션 실행 시 전송될 프롬프트 구문 ]        |
| +------------------------+ +--------------------------------------------------+ |
| | Add Snippet to Chat    | | 액션명: [ Review Code Quality ]                    | |
| | Explain Code Semantics | | 단축키: [ ^ + Opt + R         ] (바인딩)           | |
| | Review Code Quality ◀ | |                                                  | |
| | Refactor Code Snippet  | | [프롬프트 스크립트 에디터]                         | |
| | Inspect Static Analysis| | 제공된 코드 블록을 분석하여 잠재적 버그, 가독성, | |
| | Analyze Impact Scope   | | 예외 처리 누락 등을 평가하세요. 코드 스타일은    | |
| | Generate Unit Tests    | | SonarQube 표준 룰을 엄격하게 적용하십시오.       | |
| | Scan Security Vulner...| | 위반 사항이 발견되면 반드시 `Diff View` 마크업을 | |
| +------------------------+ | 이용하여 교정된 코드를 제안해야 합니다.          | |
|                            +--------------------------------------------------+ |
|                                                                                 |
|                                                     [ Apply ] [ OK ] [ Cancel ] |
+---------------------------------------------------------------------------------+
```
